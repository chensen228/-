Cypher 查询范例

系统会自动导出可执行的 Cypher 脚本，以下语句用于展示图模型中的典型查询方式。

MATCH (s:Student {studentNo: '20230002'})-[:ENROLLED_IN]->(c:Course)<-[:ENROLLED_IN]-(peer:Student)-[:ENROLLED_IN]->(rec:Course) WHERE NOT (s)-[:ENROLLED_IN]->(rec) RETURN rec.courseName, count(DISTINCT peer) AS support ORDER BY support DESC LIMIT 5
MATCH (s:Student {studentNo: '20230002'})-[:BORROWED]->(:Book)-[:IN_CATEGORY]->(cat:Category)<-[:IN_CATEGORY]-(book:Book) WHERE NOT (s)-[:BORROWED]->(book) RETURN cat.name, book.title LIMIT 5
MATCH p = (s:Student {studentNo: '20230002'})-[:HAS_TASK]->(:Task)-[:GUIDED_BY|AT_BASE]->() RETURN p LIMIT 5

当前图谱运行方式

图谱模块默认采用本地图分析与 Cypher 导出双轨模式，便于在单机环境中直接运行。

本地图分析

当前页面使用本地图模型完成推荐、路径分析和中心性计算，不依赖外部图数据库即可展示关系逻辑。

Cypher 导出

系统会自动生成 `smart_campus_graph.cypher`，并支持在上方一键复制导出路径，便于后续把相同节点和关系导入 Neo4j。

统一业务键

学生学号、课程编码、图书分类和项目任务等业务键会直接映射到图节点与关系中，减少重复建模成本。

Neo4j 切换说明

图谱逻辑已经和连接方式解耦，后续如需接入 Neo4j，只需要补充连接配置即可。

配置入口

系统支持从环境变量或 `neo4j_runtime.json` 读取连接地址、用户名和密码。

切换结果

连接可用时会从本地图分析模式切换到 Neo4j 在线模式，并直接把 Cypher 脚本同步执行到图数据库中。

当前状态

Neo4j 已连接（bolt://127.0.0.1:7687）。这种设计适合单机演示，也便于后续迁移到真实图数据库集群。

技术结构：SQLite 负责事务层，Redis 负责缓存与排行，MongoDB 负责日志、画像和文档扩展，图谱模块负责关系抽象与 Cypher 导出。
