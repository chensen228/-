跨库一致性策略

系统在跨 SQLite、Redis 和 MongoDB 写入时，优先保证事务层正确，再同步刷新缓存与文档留痕。

事务主库优先

借阅、选课、预约等核心写操作先落到 SQLite，确保库存、容量和资源占用等关键状态正确。

缓存与文档后续同步

事务成功后再更新 Redis 排行榜、配额摘要，并写入 MongoDB 行为日志、变更记录和画像文档。

补偿与重建

系统保留从事务数据重建 Redis 排行和摘要的能力，适合解释最终一致性与缓存补偿策略。

数据分层与并发扩展

除了当前演示结构，系统也预留了面向更大数据量和更高并发场景的演进方向。

冷热数据分层

Redis 保存日常热点数据，MongoDB 保存近阶段过程文档，历史日志可继续归档到 HDFS 或对象存储。

消息队列削峰

在抢课等高并发场景中，可在 Web 层和事务层之间增加 Kafka 或 RabbitMQ，对请求进行排队与限速消费。
